Respostas

  1. Inicia-se um comentário com (!–) e fecha com (–>). Os comentários podem ter muitas ou apenas uma linha.
  2. Para criar um hiperlink escolha o texto ou a imagem que você deseja exibir como um hiperlink. Pressione Ctrl+K. Você também pode clicar com o botão direito do mouse no texto ou imagem e clicar em Link no menu de atalho.
  3. O Elemento _HTML (abbr) _(ou Elemento de Abreviação HTML) representa uma abreviação e opcionalmente fornece uma descrição completa para ela. Se presente, o atributo title deve conter a descrição completa e apenas ela. Use o atributo title para definir a descrição completa da abreviação. Exemplo: (p)Obama é presidente dos (abbr title="Estados Unidos da América")EUA(/abbr)(/p) Resultado: Obama e presidente dos EUA.
  4. A tag (address) em HTML é usada para escrever informações de contato de uma pessoa ou organização. Se a tag (address) for usada dentro da tag (body), ela representa as informações de contato do documento e se a tag (address) for usada dentro da tag (article), ela representa as informações de contato do artigo. O texto dentro da tag (address) será exibido em formato itálico. Algum navegador adicionando uma quebra de linha antes e depois do elemento de endereço.
  5. A abordagem deste artigo é criar uma área dentro de um mapa de imagem usando uma tag em um documento HTML. É usado para direcionar o usuário a diferentes links depois que o usuário clica nas partes mapeadas da imagem. É usado como uma tag filha da tag . Não tem tag de fechamento e, portanto, é uma tag vazia para HTML.

  10. A URL especifica o local de um destino armazenado em um computador local ou em rede, como um arquivo, um diretório, uma página HTML, uma imagem, um programa etc. Nesta discussão, a URL absoluta terá este formato: (scheme://server/path/resource). A URL relativa localiza um recurso usando uma URL absoluta como ponto inicial. Na verdade, a "URL completa" do destino é especificada concatenando as URLs absoluta e relativa. Em geral, uma URL relativa consiste apenas no path e, opcionalmente, no resource, mas não no scheme ou no server.
